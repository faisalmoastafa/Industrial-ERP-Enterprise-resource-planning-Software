SECTION A: CORE MANUFACTURING ARCHITECTURE & BUSINESS FLOW PARADIGM
================================================================================

BUSINESS IDENTITY
-----------------
NECI (Super and Cables Industries) operates as an industrial recycling and
remanufacturing enterprise. The ERP manages the full lifecycle from raw scrap
material procurement through to finished product sales.

MATERIAL FLOW MATRIX
--------------------

STAGE 1 — RAW MATERIAL PROCUREMENT (Weight-Based / Kilograms)
  Input sources: ship generators, transformers, damaged wiring coils, copper
  scrap, aluminium scrap, and other industrial recyclables.
  Tracked by: product_quantity (DECIMAL kg) in products table.
  Product type flag: product_type = 'raw_material'
  Purchase flow: Purchases module → purchase_items → increments product_quantity

STAGE 2 — MANUFACTURING / STRIPPING PROCESS (Weight Deduction Engine)
  The ProductionBatchController.store() method executes a DB::transaction():
  - Locks raw material rows with lockForUpdate()
  - Validates sufficient kg stock availability
  - Deducts raw kg from product_quantity
  - Calculates raw_material_cost = quantity × unit_cost
  - Calculates wastage_weight = input_weight - output_weight
  - Computes cost_per_output_kg = total_cost / output_weight
  Tables: production_batches, production_batch_inputs, production_batch_outputs

STAGE 3 — FINISHED PRODUCT OUTPUT (Count-Based / Pieces)
  Output products: remanufactured cables, wire spools, copper rods (by Pcs)
  Product type flag: product_type = 'finished'
  The engine increments product_quantity (Pcs) using weighted average costing:
    new_avg_cost = ((current_qty × current_cost) + (new_qty × cost_per_kg))
                   / (current_qty + new_qty)

STAGE 4 — SALES (Count-Based / Pieces → Revenue)
  Sales module processes finished products by Pcs count.
  Decrements product_quantity on sale_items insert.
  Generates profit tracking via sale total vs. product_cost baseline.

KEY DATABASE TABLE RELATIONSHIPS
---------------------------------
products              → Central inventory. product_type distinguishes raw/finished.
production_batches    → Batch header: date, status, weights, costs.
production_batch_inputs  → Raw kg consumed per batch (FK: production_batch_id, product_id)
production_batch_outputs → Finished Pcs produced per batch (FK: production_batch_id, product_id)
production_batch_expenses → Conversion costs attached to a batch
purchases / purchase_items → Restocks raw material kg
sales / sale_items         → Depletes finished product Pcs

C.3 — INDUSTRIAL WEIGHT-TO-UNIT MATERIAL DEDUCTION ENGINE
         File: Modules/Manufacturing/Http/Controllers/ProductionBatchController.php
         Method: store() — DB::transaction() block
--------------------------------------------------------------------------------

DB::transaction(function () use ($validated) {

    // 1. Lock all involved product rows to prevent race conditions
    $products = Product::whereIn('id', $productIds)->lockForUpdate()->get()->keyBy('id');

    $inputWeight = 0; $rawMaterialCost = 0; $outputWeight = 0; $conversionCost = 0;

    // 2. PRE-FLIGHT: Validate raw material availability and accumulate input totals
    foreach ($validated['inputs'] as $row) {
        $product  = $products[$row['product_id']];
        $quantity = (float) $row['quantity'];

        if (!$product->isRawMaterial()) {
            throw ValidationException::withMessages([
                'inputs' => "{$product->product_name} is not marked as a raw material product.",
            ]);
        }

        if ((float) $product->product_quantity < $quantity) {
            throw ValidationException::withMessages([
                'inputs' => "Not enough stock for {$product->product_name}. Available: {$product->product_quantity}",
            ]);
        }

        $unitCost = (isset($row['unit_cost']) && $row['unit_cost'] !== '')
            ? (float) $row['unit_cost']
            : (float) $product->product_cost;

        $inputWeight     += $quantity;
        $rawMaterialCost += $quantity * $unitCost;
    }

    // 3. Validate output weight does not exceed input weight
    foreach ($validated['outputs'] as $row) {
        $outputWeight += (float) $row['quantity'];
    }
    if ($outputWeight > $inputWeight) {
        throw ValidationException::withMessages([
            'outputs' => 'Finished output cannot be greater than raw material input.',
        ]);
    }

    // 4. Calculate cost metrics
    $totalCost        = $rawMaterialCost + $conversionCost;
    $costPerOutputKg  = $totalCost / $outputWeight;

    // 5. Create batch header record
    $batch = ProductionBatch::create([
        'date'             => $validated['date'],
        'status'           => 'Completed',
        'input_weight'     => $inputWeight,
        'output_weight'    => $outputWeight,
        'wastage_weight'   => $inputWeight - $outputWeight,
        'raw_material_cost'=> $rawMaterialCost,
        'conversion_cost'  => $conversionCost,
        'total_cost'       => $totalCost,
        'cost_per_output_kg' => $costPerOutputKg,
        'user_id'          => auth()->id(),
        'completed_at'     => now(),
    ]);

    // 6. DECREMENT raw material kg stock
    foreach ($validated['inputs'] as $row) {
        $product  = $products[$row['product_id']];
        $quantity = (float) $row['quantity'];
        ProductionBatchInput::create([...]);
        $product->update([
            'product_quantity' => (float) $product->product_quantity - $quantity,
        ]);
    }

    // 7. INCREMENT finished product Pcs stock with weighted average cost
    foreach ($validated['outputs'] as $row) {
        $product         = $products[$row['product_id']];
        $quantity        = (float) $row['quantity'];
        $currentQuantity = (float) $product->product_quantity;
        $currentCost     = (float) $product->product_cost;
        $newQuantity     = $currentQuantity + $quantity;
        $newAverageCost  = $newQuantity > 0
            ? (($currentQuantity * $currentCost) + ($quantity * $costPerOutputKg)) / $newQuantity
            : $costPerOutputKg;

        ProductionBatchOutput::create([...]);
        $product->update([
            'product_quantity' => $newQuantity,
            'product_cost'     => $newAverageCost,
        ]);
    }

    // 8. Security audit log
    Log::channel('security')->info('Production batch completed', [
        'user_id'             => auth()->id(),
        'production_batch_id' => $batch->id,
        'input_weight'        => $inputWeight,
        'output_weight'       => $outputWeight,
        'wastage_weight'      => $inputWeight - $outputWeight,
        'total_cost'          => $totalCost,
    ]);
});

The application runs on XAMPP local stack. For desktop packaging, the internal
URL must point to the local PHP server, NOT the XAMPP Apache virtual host.

REQUIRED .env SETTINGS FOR DESKTOP MODE:
  APP_URL=http://127.0.0.1:8000
  APP_ENV=local
  APP_DEBUG=false          ← Set to false for production desktop build
  SESSION_SECURE_COOKIE=false
  SESSION_SAME_SITE=lax

The Nativefier wrapper will load: http://127.0.0.1:8000
The PHP artisan serve command must be running in the background.

--------------------------------------------------------------------------------
3.2 — NATIVEFIER COMPILATION RECIPE
--------------------------------------------------------------------------------

PREREQUISITES:
  1. Node.js installed (https://nodejs.org — LTS version)
  2. npm available in PATH

STEP 1 — Install Nativefier globally:
  npm install -g nativefier

STEP 2 — Prepare the application icon (optional):
  Place a 256x256 PNG icon at: C:\xampp\htdocs\neci-erp\public\images\logo.png
  (Already exists in the project)

STEP 3 — Start the local server (must be running before packaging):
  c:\xampp\php\php.exe artisan serve --host=127.0.0.1 --port=8000

STEP 4 — Run the Nativefier compilation command:
  Open a NEW terminal window and run:

  nativefier ^
    --name "NECI ERP System" ^
    --app-version "2.0.0" ^
    --width 1280 ^
    --height 800 ^
    --min-width 1024 ^
    --min-height 768 ^
    --internal-urls ".*" ^
    --inject "neci-desktop.css" ^
    --hide-window-frame false ^
    --disable-context-menu ^
    --disable-dev-tools ^
    --single-instance ^
    --tray ^
    --icon "C:\xampp\htdocs\neci-erp\public\images\logo.png" ^
    --user-agent "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36" ^
    --no-overwrite ^
    --platform win32 ^
    --arch x64 ^
    "http://127.0.0.1:8000" ^
    "C:\NECI-ERP-Desktop"

STEP 5 — Output location:
  The compiled .exe will be at: C:\NECI-ERP-Desktop\NECI ERP System-win32-x64\
  Executable: "NECI ERP System.exe"

STEP 6 — Create a desktop shortcut pointing to "NECI ERP System.exe"

FLAG EXPLANATIONS:
  --internal-urls ".*"     → All URLs stay inside the app window (no browser pop-outs)
  --single-instance        → Only one instance of the app can run at a time
  --disable-dev-tools      → Hides F12 developer tools from end users
  --disable-context-menu   → Removes right-click browser menu
  --tray                   → App minimizes to system tray instead of closing
  --no-overwrite           → Prevents accidental rebuild overwriting existing install

NECI ERP 2.0 — LAUNCH INSTRUCTIONS
=====================================

The NECI ERP runs as a local web application wrapped in a desktop window.
You must start the server BEFORE opening the application.

METHOD A — AUTOMATIC LAUNCH (Recommended)
Use the provided start-neci-erp.bat launcher script:
  Double-click: C:\xampp\htdocs\neci-erp\start-neci-erp.bat
  This will:
    1. Start XAMPP Apache and MySQL services
    2. Start the PHP application server on port 8000
    3. Open the NECI ERP desktop application

METHOD B — MANUAL LAUNCH (Step by Step)

STEP 1 — Start XAMPP Services:
  Open XAMPP Control Panel: C:\xampp\xampp-control.exe
  Click [Start] next to Apache
  Click [Start] next to MySQL
  Wait for both to show green status

STEP 2 — Start the PHP Application Server:
  Open Command Prompt (cmd.exe)
  Run: c:\xampp\php\php.exe c:\xampp\htdocs\neci-erp\artisan serve --host=127.0.0.1 --port=8000
  Leave this window OPEN (minimise it, do not close)

STEP 3 — Launch the Application:
  Double-click the "NECI ERP System" desktop shortcut
  OR run: C:\NECI-ERP-Desktop\NECI ERP System-win32-x64\NECI ERP System.exe

STEP 4 — Login:
  Email:    adminfaisal@neci.com
  Password: necishop

TO CLOSE THE APPLICATION:
  Close the NECI ERP window
  In the Command Prompt from Step 2, press Ctrl+C to stop the server
  In XAMPP Control Panel, click [Stop] for Apache and MySQL

TROUBLESHOOTING:
  "This site can't be reached" → PHP server is not running. Repeat Step 2.
  "Database error"             → MySQL is not running. Repeat Step 1.
  "Login failed"               → Check credentials above.
  "Port 8000 in use"           → Change --port=8001 in Step 2 and update APP_URL in .env

@echo off

echo.

echo Close this window to keep the server running in background.
